# 第5章: AOP とロギング（REST API運用向け）

この章では、API運用で重要な「誰が・いつ・どのAPIを・どれくらいの時間で実行したか」を追跡できるようにします。

---

## 5.1 依存関係

`pom.xml` に AOP を追加（未追加の場合）。

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>
```

---

## 5.2 Aspectの基本方針

全パッケージを雑に対象化すると、Securityやフレームワーク処理に過剰介入しやすくなります。  
まずは `api` と `service` を中心にしつつ、監視向けの `health` だけを明示的に追加するのが安全です。

```java
@Around(
    "execution(* com.example.demo.api..*.*(..)) || " +
    "execution(* com.example.demo.service..*.*(..)) || " +
    "execution(* com.example.demo.HealthController.*(..))"
)
public Object logAround(ProceedingJoinPoint pjp) throws Throwable {
    var signature = pjp.getSignature().toShortString();
    long start = System.currentTimeMillis();
    try {
        Object result = pjp.proceed();
        long elapsed = System.currentTimeMillis() - start;
        log.info("API OK: {} ({} ms)", signature, elapsed);
        return result;
    } catch (Exception ex) {
        long elapsed = System.currentTimeMillis() - start;
        log.error("API NG: {} ({} ms) {}", signature, elapsed, ex.getMessage());
        throw ex;
    }
}
```

---

## 5.3 実運用で追加したいログ

- リクエストID（トレーサビリティ）
- 認証ユーザーID
- HTTPメソッド/パス
- ステータスコード
- 主要ドメインキー（受注番号など）

ログに機密情報（パスワード、トークン、生データ）を出さないルールも必須です。

---

## 5.4 確認ポイント

- API呼び出しごとに実行時間が出る
- 例外発生時に異常ログが残る
- Security設定に悪影響がない（認証系テストが壊れない）
