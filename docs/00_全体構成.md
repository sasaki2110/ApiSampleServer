# 全体構成（REST API 前提版）

この教材は、次の構成を前提にしています。

- フロントエンド: Vue.js + ag-grid（別リポジトリ/別プロセス）
- バックエンド: Spring Boot 4.0.4 + Java 25
- 連携方式: JSON REST API
- 方針: Thymeleaf は利用しない

---

## 学習ロードマップ

### 第1章: Spring Boot API 基礎と環境構築
- **Topic:** 20年前のMVCから、現代のAPI中心アーキテクチャへの移行
- **Hands-on:** Spring Initializr、`pom.xml` 整理、`@RestController` で疎通確認
- **Goal:** `GET /api/health` が200で返る最小APIを作る

### 第2章: H2 Database と Spring Data JPA
- **Topic:** エンティティ設計、Repository、トランザクション境界
- **Hands-on:** `Project` エンティティ作成、CRUD API実装、H2確認
- **Goal:** DB永続化付きAPIを動かす

### 第3章: REST 設計（DTO / Validation / 例外）
- **Topic:** ControllerにEntityを直接返さない設計、入力検証、エラー標準化
- **Hands-on:** Request/Response DTO、`@Valid`、`@RestControllerAdvice`
- **Goal:** フロントが扱いやすいAPI仕様を作る

### 第4章: Spring Security（API向け）
- **Topic:** 画面向け設定ではなく、API向け認証・認可の基本
- **Hands-on:** `SecurityFilterChain`、エンドポイント公開範囲、CORS/CSRF方針
- **Goal:** Vueクライアントと安全に接続できるAPI基盤を作る

### 第5章: AOP とロギング
- **Topic:** 横断関心事（監査ログ、実行時間計測）の分離
- **Hands-on:** Controller/Service層の共通ログ出力
- **Goal:** 業務コードを汚さず可観測性を上げる

### 第6章: JUnit 5 / Spring Boot Test
- **Topic:** APIテストの基本、Security込みテスト、Repositoryテスト
- **Hands-on:** `MockMvc`、`@DataJpaTest`、異常系テスト
- **Goal:** 変更に強いバックエンドを作る

### 第7章: 次ステップ（外部連携・運用）
- **Topic:** OpenFeign、Resilience4j、Docker、CI
- **Hands-on:** 外部API連携と障害耐性の導入
- **Goal:** 実運用を見据えた拡張方針を持つ

---

## この教材で重視すること

- API契約（入力/出力/エラー）の一貫性
- サーバ側での検証と整合性担保
- フロント実装に依存しすぎないバックエンド設計
- 将来の本番DB移行を見据えた実装
