# 次ステップへの展望（Vue + Spring REST）

第1章から第6章までを終えたら、次は実務向けに拡張します。

---

## 1. フロント連携の本格化

- Vue + ag-grid からの入力仕様をAPI契約に反映
- バージョン付きAPI（例: `/api/v1/...`）の導入
- OpenAPI（Swagger）で契約を明文化

---

## 2. ドメイン強化

- 受注ヘッダ/受注明細のトランザクション設計
- サーバ側再計算（数量 x 単価 = 金額）で整合性担保
- 楽観ロック（`@Version`）で同時更新競合を防止

---

## 3. インフラ・運用強化

- H2から本番DB（PostgreSQLなど）への移行
- Flyway/Liquibaseでスキーマ管理
- Docker化、CIで `./mvnw test` を自動実行

---

## 4. 外部連携と耐障害性

- OpenFeignで外部マスタ連携
- Resilience4jでタイムアウト/リトライ/サーキットブレーカー
- 障害時のフォールバック設計

---

## 5. 到達イメージ

最終的には、次を満たす状態を目指します。

- Vue側は入力体験と画面制御に集中
- Spring側は検証・整合性・監査・永続化を担当
- API契約が安定し、フロント/バックの分業がスムーズ
